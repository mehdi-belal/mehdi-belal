# Profile README Setup

This repository is a special GitHub profile repository.

When a repository name matches the GitHub username, the root `README.md` is rendered on the profile page:

- GitHub profile repo pattern: `<username>/<username>`
- Profile page source: `README.md`

## How It Works

The profile page is a mix of static content and generated assets.

Static content:

- social links in `README.md`
- technology badges in `README.md`
- section heading SVG files in `svg/`

Dynamic content:

- `profile/top-langs.svg`
- `profile/stats.svg`

Important:

- the current `README.md` only displays `profile/top-langs.svg`
- `profile/stats.svg` is still generated by CI, but it is not currently used in the README

## Current Repository Structure

- `README.md`: the profile page shown on GitHub
- `.github/workflows/update_cards.yml`: generates the GitHub stats cards
- `.github/workflows/codeberg-sync.yml`: syncs the repository to Codeberg
- `profile/`: generated SVG cards
- `svg/`: local section title SVG files used by the README
- `docs/`: documentation for maintaining the repository

## README Editing Rules

Edit `README.md` when you want to change:

- social links
- badges
- ordering of sections
- which generated assets are displayed

Edit files in `svg/` when you want to change:

- the section title text
- the section title typography
- the SVG heading style

Do not edit files in `profile/` by hand unless you intentionally want to override generated output. The workflow may replace them on the next run.

## Dynamic Card Workflow

File:

- `.github/workflows/update_cards.yml`

What it does:

1. Checks out the repository.
2. Ensures the `profile/` folder exists.
3. Generates `profile/stats.svg`.
4. Generates `profile/top-langs.svg`.
5. Commits and pushes the generated files if they changed.

When it runs:

- every day at `00:00 UTC`
- manually from the GitHub Actions tab

Current behavior:

- both cards are generated
- only `profile/top-langs.svg` is shown in `README.md`

If you want to stop generating the unused stats card, remove the `Generate stats card` step from `.github/workflows/update_cards.yml`.

## Codeberg Sync

File:

- `.github/workflows/codeberg-sync.yml`

What Codeberg is:

- Codeberg is a Git hosting platform, similar to GitHub
- it is commonly used as a mirror or secondary remote

What this workflow is for:

- keep a copy of this repository on Codeberg
- mirror README, workflows, SVG files, and all other tracked repository files

What you need before it can work:

1. a Codeberg account
2. a Codeberg access token
3. a GitHub repository secret named `CODEBERG_TOKEN`
4. a target repository on Codeberg, or a workflow that can create it

## How To Set Up Codeberg

### 1. Create a Codeberg account

Sign up at:

- `https://codeberg.org`

### 2. Create a Codeberg access token

Create the token here:

- `https://codeberg.org/user/settings/applications`

Recommended token permissions for mirroring:

- `write:repository`

If the sync workflow also needs to create repositories automatically, you may also need:

- `write:user`
- `write:organization`

### 3. Add the token to GitHub

In GitHub:

1. Open the repository.
2. Go to `Settings`.
3. Open `Secrets and variables` > `Actions`.
4. Create a new repository secret.
5. Name it `CODEBERG_TOKEN`.
6. Paste the Codeberg token value.

### 4. Create the target Codeberg repository

Recommended repository name:

- use the same repository name as the source repository unless you intentionally want a different mirror name

Recommended owner:

- use your Codeberg username or organization name

If you create it manually, create an empty repository. Do not initialize it with:

- `README`
- `.gitignore`
- `LICENSE`

That avoids unrelated-history issues on first mirror.

## Running The Workflows

### Update cards manually

1. Open the GitHub repository.
2. Go to `Actions`.
3. Open `Update README cards`.
4. Click `Run workflow`.

Result:

- the generated SVG files under `profile/` are refreshed

### Run Codeberg sync manually

1. Open the GitHub repository.
2. Go to `Actions`.
3. Open `Codeberg sync`.
4. Click `Run workflow`.

Result:

- the repository is pushed or mirrored to Codeberg

## Troubleshooting

### `CODEBERG_TOKEN` secret is missing

Symptom:

- the Codeberg sync workflow fails early

Fix:

- create the GitHub Actions secret named `CODEBERG_TOKEN`

### `repository not found` during Codeberg push

Symptom:

- the workflow reaches Codeberg but fails to push

Common causes:

- the Codeberg repository does not exist
- the Codeberg username or repository name is wrong
- the token does not have enough permission

Fix:

1. confirm the target Codeberg username
2. confirm the target repository exists
3. confirm the token belongs to the correct Codeberg account
4. confirm the token has at least `write:repository`

### Top languages card is broken in the README

Symptom:

- broken image in the profile README

Common causes:

- `profile/top-langs.svg` was not generated yet
- the workflow failed
- the file path in `README.md` is wrong

Fix:

1. run `Update README cards` manually
2. confirm `profile/top-langs.svg` exists in the repository
3. confirm `README.md` points to `./profile/top-langs.svg`

## Maintenance Notes

- Keep `README.md` as the single source of truth for profile layout.
- Keep generated files in `profile/`.
- Keep local handcrafted SVG section titles in `svg/`.
- If you remove a generated asset from the README, consider also removing its generation step from CI.
